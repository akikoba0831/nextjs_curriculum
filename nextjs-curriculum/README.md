# Next.js 学習カリキュラム(全10日)

React学習済みの人向けに、Next.js (App Router) を実際に手を動かしながら学ぶためのカリキュラムです。

## 進め方

1. 各 `dayXX_xxx` フォルダの中の `README.md` を開き、その日の課題を読む
2. **📚 解説を先に読み、「なぜそう書くのか」を理解してから**実施タスクに進む(写経するだけでは応用が利く知識になりません)
3. 実際にコードを書いて動かす
4. 🧠 理解度チェックの設問に自分の言葉で答えられるか確認する
5. 完了チェックリストを確認してから次のDayに進む

⚠️ 注意: Day1で作る **1つのNext.jsプロジェクト** に対して、Day2以降も追記・変更していきます。
Dayごとに別プロジェクトを作るわけではありません。各フォルダのREADMEは「その日にやることの説明書」だと思ってください。

## 全体スケジュール

| Day | フォルダ名 | テーマ |
|---|---|---|
| 1 | day01_setup | 環境構築 & プロジェクト作成 |
| 2 | day02_app_router | App Routerの基本(ページ・リンク) |
| 3 | day03_server_client_components | Server / Client Componentsの使い分け |
| 4 | day04_data_fetching | データ取得とLoading UI |
| 5 | day05_dynamic_routing | 動的ルーティング([id]) |
| 6 | day06_layout_metadata | レイアウトとMetadata(SEO) |
| 7 | day07_forms_server_actions | フォームとServer Actions |
| 8 | day08_api_routes | API Routes (Route Handlers) |
| 9 | day09_styling | Tailwind CSSでのスタイリング |
| 10 | day10_deploy | Vercelへのデプロイ |

## 前提知識
- React (useState, useEffect, props, コンポーネント) は理解済み

## 必要なツール
- Node.js (18以上推奨)
- VSCode
- ターミナル操作の基本
