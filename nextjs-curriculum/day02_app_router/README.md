# Day 2: App Routerの基本

## 🎯 目標
ファイルを置くだけでページが増える「ファイルベースルーティング」を理解する

## 📖 学ぶ内容
- `app` フォルダの中にファイルを作ってページを追加する方法
- `next/link` を使ったページ遷移
- `<a>` タグとの違い

## 📚 解説

### ファイルシステムベースルーティング
App Routerでは「フォルダ構造 = URL構造」です。設定ファイルにルートを列挙する必要はなく、`app/about/page.tsx` を作れば自動的に `/about` というURLが生まれます。

重要なのは、**フォルダを作っただけではURLは公開されない**という点です。フォルダの中に`page.tsx`という名前のファイルが存在して初めて、そのセグメントが外部からアクセス可能なページになります。逆に言うと、`page.tsx`を持たないフォルダ(例えば後の Day で扱う `components/` のような置き場所)を`app`の中に作っても、それはURLにはなりません。

### なぜ `next/link` は `<a>` より速いのか
`<a href="/about">` でページ遷移すると、ブラウザは新しいページを**フルロード**します。これはHTML/CSS/JSを最初から全部読み直す処理で、Reactの状態やスクロール位置なども含めて全てリセットされます。

`next/link` の `<Link>` コンポーネントはこれを避け、**クライアントサイドナビゲーション**という仕組みで遷移します。具体的には以下の2つが起きています。

1. **プリフェッチ**: `<Link>`が画面内に表示された時点で、Next.jsは遷移先のページに必要なデータを裏側で先読みしておく
2. **部分差し替え**: 実際にクリックされたら、ページ全体を読み直すのではなく、変化した部分のコンポーネントツリーだけをサーバーから取得してその部分だけ差し替える

この結果、体感的には画面が一瞬で切り替わるSPA(Single Page Application)のような挙動になりますが、実際には各ページが個別にサーバーでレンダリングされる仕組み(App Router)の上に成り立っています。「サーバーで生成されたページ同士を、遷移だけクライアントサイドで軽くする」というのがNext.jsのアプローチです。

## ✅ 実施タスク

### 1. Aboutページを作る

`app/about/page.tsx` というファイルを新規作成してください。

```tsx
export default function About() {
  return (
    <main>
      <h1>About</h1>
      <p>このサイトについての説明ページです。</p>
    </main>
  );
}
```

→ `http://localhost:3000/about` にアクセスして表示されればOK

### 2. ナビゲーションリンクを作る

`app/layout.tsx` を開き、`<body>` の中にナビゲーションを追加します。

```tsx
import Link from "next/link";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ja">
      <body>
        <nav>
          <Link href="/">Home</Link> | <Link href="/about">About</Link>
        </nav>
        {children}
      </body>
    </html>
  );
}
```

### 3. Contactページも同様に作ってみる
`app/contact/page.tsx` を自分で作成し、ナビゲーションにもリンクを追加してみましょう。

## 💡 なぜ `<a>` ではなく `next/link` を使うのか
- `<a>` タグ: ページ全体を再読み込みする(遅い)
- `next/link`: 必要な部分だけ差し替える(速い、SPAのような動き)

## 🧠 理解度チェック
1. `app/contact/page.tsx` を作らずに `app/contact/` フォルダだけを作った場合、`/contact` にアクセスするとどうなるか説明せよ。
2. `<Link>`が「プリフェッチ」をしているとして、それがユーザー体験にどう影響するか(何が速くなるか)説明せよ。
3. **予想→検証**: `<a href="/about">` と `<Link href="/about">` の両方をナビゲーションに置き、ブラウザの開発者ツールでネットワークタブを開いた状態でそれぞれをクリックしてみよ。読み込まれるリソースの量にどんな違いが出るか予想してから確認せよ。
4. `next/link` を使わず全てのページ遷移を `<a>` タグで実装した場合、ユーザー体験にどのようなデメリットがあるか説明せよ。
